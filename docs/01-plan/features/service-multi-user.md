# Plan: 서비스화 마스터 플랜 (로컬 퍼스트 & 선택적 클라우드 백업)

## 1. 개요 (Background & Motivation)
현재 앱은 1인 전용으로 개발되어 있으며, Firestore에 데이터를 직접 저장하고 있습니다. 이를 다수의 유저가 안전하게 사용할 수 있도록 개선하면서도, 신규 유저가 가입 없이 즉시 가치를 느낄 수 있도록 '로컬 퍼스트' 방식을 채택합니다. 또한 비용이 많이 발생하는 AI 서비스에 대해 지속 가능한 운영 전략을 수립합니다.

## 2. 핵심 목표 (Scope & Impact)
- **Zero-Friction 온보딩:** 로그인 없이 즉시 자산 입력 및 대시보드 사용 가능.
- **선택적 클라우드 동기화:** 설정 페이지에서 로그인 시 로컬 데이터를 Firestore로 백업 및 기기 간 동기화.
- **AI 서비스 최적화:** 비용 효율적인 AI 서비스 제공 전략(일일 제한, 로그인 필수 등).
- **데이터 보안:** 유저별 데이터 격리 및 권한 관리.

## 3. 제안된 솔루션 (Proposed Solution)

### A. 데이터 저장 아키텍처 (Local-First with Cloud Sync)
1. **기본 상태:** 모든 데이터는 `localStorage`(현재 Zustand Persist 활용)에만 저장됩니다.
2. **로그인 연동:** 사용자가 설정에서 'Google 로그인' 등을 수행하면:
   - 로컬의 최신 데이터를 Firestore의 `users/{uid}/...` 경로로 업로드(Initial Sync).
   - 이후 데이터 변경 시 로컬과 Firestore를 동시에 업데이트.
   - 다른 기기에서 로그인 시 Firestore 데이터를 로컬로 다운로드.
3. **Firestore 보안 규칙:** `request.auth.uid`를 기반으로 본인의 데이터만 접근 가능하도록 설정.

### B. AI 서비스 제공 전략 (AI Service Strategy)
AI는 비용이 많이 발생하므로 다음과 같은 정책을 검토합니다:
1. **무료 유저 (비로그인):**
   - **AI 어드바이저:** 사용 불가 또는 매우 제한적인 일일 횟수(예: 1회).
   - **AI 인사이트:** 최근 24시간 내 캐시된 정보만 제공(신규 생성 불가).
2. **인증 유저 (로그인):**
   - **AI 어드바이저:** 일일 5~10회 질문 가능.
   - **AI 인사이트:** 자산 변경 시 일일 최대 3회 자동 갱신.
3. **기술적 최적화:** 
   - GPT-4o-mini 등 저비용 모델 우선 활용.
   - 중복 질문 방지를 위한 서버 사이드 캐싱 강화.

## 4. 단계별 구현 계획 (Implementation Plan)

### Phase 1: 인증 시스템 및 UI 준비
- Firebase Auth(Google Provider) 활성화.
- 설정(`settings/page.js`) 내 '클라우드 백업 및 동기화' 섹션 추가 (로그인 버튼 포함).

### Phase 2: 클라우드 동기화 엔진 개발
- `useAssetStore` 내 `syncWithCloud` 함수 구현.
- 로컬 데이터와 서버 데이터 간의 충돌 해결 로직(Timestamp 기반) 수립.
- 기존 API(`/api/assets` 등)를 유저 UID 기반으로 리팩토링.

### Phase 3: AI 권한 및 비용 제어 시스템
- `/api/ai-*` 라우트에서 유저의 로그인 여부 및 사용 횟수(Rate Limit) 체크 로직 추가.
- 비로그인 유저를 위한 안내 UI(Upsell/Guide) 구현.

### Phase 4: 보안 및 안정성 강화
- Firestore Security Rules 배포.
- 데이터 내보내기/삭제(탈퇴) 등 유저 권리 보호 기능 추가.

## 5. 검증 및 테스트 (Verification)
- **비로그인 시나리오:** 데이터가 `localStorage`에만 정상 저장되는지 확인.
- **로그인 시나리오:** 로컬 데이터가 Firestore로 성공적으로 전송되는지 확인.
- **다중 기기 시나리오:** 기기 A에서 변경한 내용이 기기 B(로그인 시)에 반영되는지 확인.
- **AI 제한 시나리오:** 일일 횟수 초과 또는 비로그인 시 적절한 안내가 나가는지 확인.

## 6. 대안 고려사항 (Alternatives Considered)
- **완전 로컬 방식:** 동기화가 불가능하여 기기 변경 시 데이터 유실 위험.
- **강제 로그인 방식:** 신규 유저 이탈률이 높음.

## 7. 수익화 모델 (Monetization Strategy) - 세계 최고의 기획자 관점 고도화
단순한 기능 제한을 넘어, 유저가 '내 자산이 소중해질수록' 자연스럽게 구독을 선택하게 만드는 **'심리적 가치 보존'** 전략을 적용합니다.

### A. 요금제 및 기능 차별화 (Tiered Value Proposition)
1. **Free (The Explorer): 가치 체감형 무전환 레이아웃**
   - **데이터:** **순수 로컬 저장(Local-only).** 브라우저 캐시 삭제 시 데이터 유실 위험을 수시로 '부드럽게' 경고하여 백업의 필요성을 인지시킴.
   - **자산 개수 제한:** **최대 5개 종목/자산까지만 등록 가능.** (6번째 자산을 등록하려는 순간 "소중한 포트폴리오를 클라우드에 안전하게 보관하고 더 확장하세요"라는 메시지와 함께 구독 유도)
   - **AI:** **맛보기 수준.** AI 인사이트는 1주일에 1회 업데이트, AI 어드바이저는 한 달에 3회 질문 가능. (AI의 답변 끝부분에 "Pro 플랜에서는 더 정교한 리밸런싱 전략을 즉시 제공합니다"라는 꼬리표 삽입)
   - **기능:** 대시보드 기본 조회.

2. **Pro (The Smart Investor): 심리적 안전 및 성과 최적화**
   - **데이터:** **안전한 클라우드 백업 및 무제한 기기 동기화.** 로그인(Auth) 기능 활성화. "내 자산 데이터가 영구적으로 안전하다"는 심리적 안정감 제공.
   - **자산 개수 제한:** **무제한 등록.** 복잡한 포트폴리오 관리가 필요한 시점의 유저에게 해방감 제공.
   - **AI:** **Personal AI Manager.** AI 어드바이저 무제한, 자산 변동 시 즉시 AI 인사이트 자동 업데이트.
   - **기능:** Pro 전용 고급 차트(자산 배분 히스토리), 엑셀/PDF 리포트 내보내기.

### B. 결제 및 로그인 전환 경험 (Conversion UX)
- **'매몰 비용(Sunk Cost)' 활용:** 유저가 무료로 5개의 자산을 정성껏 입력했을 때, 6번째 자산 입력 시점에 구독 버튼을 노출합니다. 이미 입력한 노력이 아까워서라도 구독을 선택하게 만듭니다.
- **'데이터 보험' 메시징:** "핸드폰을 바꾸거나 브라우저를 초기화해도 내 소중한 자산 데이터는 안전할까요? 지금 10초 만에 클라우드에 보험을 드세요."라는 문구로 로그인을 유도합니다.
- **부드러운 로그인 시점:** 앱 시작 시 로그인을 강요하지 않고, '백업하기' 또는 'Pro 기능 사용하기' 버튼을 누를 때만 자연스럽게 로그인 창을 띄웁니다.
- **결제 경험:** Stripe 등을 연동하여 '1개월 무료 체험' 후 자동 결제 방식을 도입, 초기 진입 장벽을 극적으로 낮춥니다.

### C. 도입 로드맵 (Roadmap)
1. **MVP (1단계):** 로컬 저장 기반 앱 + 자산 개수 제한 기능 구현 + AI 사용 횟수 제한 로직.
2. **Beta (2단계):** Firebase Auth 연동 및 '로그인 시 백업' 기능 출시.
3. **Full Launch (3단계):** 구독 결제 연동 및 Pro 전용 고급 분석 기능 강화.
